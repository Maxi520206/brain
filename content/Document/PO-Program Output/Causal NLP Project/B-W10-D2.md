根据你的 GitHub 项目 text-classification-baseline**（基于 TF-IDF + Logistic Regression 的 IMDB 情感分析）以及 **B.CausalNLP_Bulletplan** 的学习路径，你的 Classification 策略应从“统计基准”向“因果鲁棒”演进。

以下是结合你当前进度与讲义要求的分类策略建议：

### 1. 基准对齐与规范化 | Baseline Setup (Reflecting W10)

你目前的 GitHub 已经完成了基础框架，下一步需要按照 [B1-W10-D2] 的产出要求进行规范化：

- **Reproducibility | 可复现性**：在 `src/config.py` 中固定 **3 Seeds** (例如：42, 1337, 2026)，确保每次训练的 TF-IDF 词库切分与 Logistic Regression 的初始化是一致的。
- **Split Strategy | 切分策略**：由于 IMDB 数据集较大（50,000 样本），建议采用 **Validation Set Approach | 验证集方法**：
    - Training: 20,000
    - Validation: 5,000 (用于调整超参数，如正则化系数 $C$)
    - Test: 25,000
- **Regularization | 正则化**：针对 Logistic Regression，在 `LogisticRegression(penalty='l2')` 基础上通过 Cross-Validation 寻找最优的 $C$ 值，防止模型过拟合。

### 2. 诊断性评估策略 | Diagnostic Evaluation (Reflecting W11)

你的 GitHub README 提到误判多见于“混合情感”和“微妙表达”。你需要升级评估维度：

- **Confusion Matrix Analysis | 混杂矩阵分析**：不仅仅记录 Accuracy (当前为 0.8768)，更要分析 False Positive (FP) 和 False Negative (FN) 的具体案例。
- **Confidence Interval | 置信区间**：利用 **Bootstrap | 自助法** 为你的准确率计算 95% CI。句式模板可参考：“The model accuracy is 87.7% [86.2%, 89.1%]”。
- **Capability Matrix | 能力矩阵**：参考 CheckList 论文，针对 Negation（否定句）、Intensity（程度词）等维度建立测试套件，验证模型在语义转换下的稳定性。

### 3. 快捷路径学习排查 | Shortcut Learning Audit (Reflecting W10-D3)

根据 Geirhos 的理论，你的基准模型可能依赖了“快捷路径”（Shortcuts）而非真正的语义逻辑：

- **Spurious Correlation | 伪相关排查**：检查模型是否因为某些词频高的背景词（如特定演员名、导演名）而误判情感，这属于 **Texture-like Bias** 在文本中的体现。
- **Distribution Shift Test | 分布偏移测试**：将测试集按“评论长度”或“词频”分桶（Bucketing），观察模型在不同长度评论上的表现差异（ID vs. Slight Shift）。

### 4. 向因果架构演进 | Evolution to Causal Framework (Reflecting W13-W15)

长期策略应从关联发现转向机制建模：

- **Representation Upgrade | 表示升级**：从 **Bag-of-Words** 演进到 **Transformer/BERT**。正如你 README 所述，线性模型缺乏 Contextual Understanding，而 BERT 能通过 Attention 捕获统计依赖。
- **Causal Skeleton | 因果骨架**：在 W13 阶段，你需要为 IMDB 任务画出第一个 **DAG (Directed Acyclic Graph)**：
    - **Unit**: 电影评论。
    - **Treatment ($T$)**: 评论中的特定关键词（如“brilliant”）。
    - **Outcome ($Y$)**: 情感标签。
    - **Confounder ($Z$)**: 电影的主题或题材（Topic），它可能同时影响作者用词和最终评价。
