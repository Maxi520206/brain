# Observational Data and the Limits of Causal Identification
`Topic: Causal Inference`
`Category: Foundations`

Research note on the distinction between observational correlations and causal mechanisms.
## 1. The limitation of observational data

In many machine learning settings, models are trained purely from observational data.

Given a dataset containing variables X and Y, the goal is typically to learn the conditional distribution

`P(Y | X)`

or the joint distribution

`P(X, Y)`

However, observational distributions alone do not uniquely determine the underlying causal structure.

Different causal models can produce exactly the same joint distribution.

Consider two possible data-generating processes.

Model 1: Causal direction
`Y â†’ X`

Here the label Y causes the observed variable X.

Example:

A person is instructed to write a specific digit Y, which leads to a generated image X.

Model 2: Common cause
`Z â†’ X`
`Z â†’ Y`

Here both X and Y are generated by a latent variable Z.

Example:

An internal intention Z determines both the label and the written image.

Although the causal structures are different, both models may generate the same observational distribution

`P(X, Y)`

This phenomenon is known as observational equivalence.

Therefore:

Observational data alone cannot identify causal structure.

## 2. Structural Causal Models and generative mechanisms

Structural Causal Models (SCMs) provide a framework for describing how variables are generated.

Instead of defining relationships purely through probabilities, SCMs specify structural assignments:

`X := f_X(Pa_X, Îµ_X)`

where

`Pa_X` are the parent variables of X in the causal graph

`f_X` is a structural function representing a mechanism

`Îµ_X` is an exogenous noise variable

Each variable is therefore generated by a mechanism, rather than by a purely statistical dependency.

In this perspective:

Causality describes the mechanisms that generate data.

## 3. Observational equivalence

Because observational distributions only reflect correlations between variables, they often fail to uniquely determine the causal graph.

Two distinct SCMs can induce the same joint distribution:

`P(X, Y)`

For example:

`Y â†’ X`

and

`Z â†’ X`
`Z â†’ Y`

may both produce the same correlation pattern between X and Y.

Thus, observing X and Y is insufficient to determine whether:

`X` causes `Y`

`Y` causes `X`

a third variable causes both

This limitation motivates the need for intervention.

## 4. Intervention and the do-operator

In causal inference, intervention refers to actively modifying the data-generating process.

Pearl formalized this idea using the do-operator:

`P(Y | do(X = x))`

An intervention sets the value of a variable externally, overriding its usual causal mechanism.

In an SCM, if X is originally generated by

`X := f_X(Pa_X, Îµ_X)`

then under intervention

`do(X = x)`

the structural equation is replaced by

`X := x`

This operation removes all incoming causal influences on X.

Graphically, intervention corresponds to cutting the incoming edges to the intervened variable.

## 5. Why intervention reveals causation

Intervention allows us to test whether a variable truly influences another.

Consider two hypotheses.

Hypothesis 1
`X â†’ Y`

If we intervene on X

`do(X = x)`

then the distribution of Y should change.

Hypothesis 2
`Z â†’ X`
`Z â†’ Y`

In this case, intervening on X does not affect Y, because the causal influence flows through Z.

Thus, interventions allow us to distinguish between causal structures that appear identical under purely observational data.

In this sense:

Causation is defined by how a system responds to interventions.

## 6. Intervention as a mechanism test

Intervention can be viewed as a test of the underlying mechanism.

The process typically involves:

Proposing a causal structure

Intervening on one variable

Observing how other variables respond

Evaluating whether the observed response matches the predicted mechanism

Instead of estimating

`P(Y | X)`

causal inference focuses on estimating

`P(Y | do(X))`

which captures the effect of actively changing X.

## 7. Implications for machine learning

Most machine learning models operate purely at the level of correlations.

For example, a classifier typically learns

`P(Y | X)`

from training data.

However, this does not guarantee that the model captures causal relationships.

Instead, models may rely on spurious correlations.

Example:

In a sentiment classification dataset, if negative reviews tend to be longer than positive ones, a model might use sentence length as a shortcut feature.

Such shortcuts often fail under distribution shift.

To address this limitation, causal approaches introduce interventions into the learning process, for example through:

Counterfactual Data Augmentation (CDA)

feature-level interventions

environment-based training methods (IRM, GroupDRO)

causal probing of representations

These approaches attempt to evaluate whether model predictions remain stable when potential spurious features are modified.

## 8. Key takeaway

Observational correlations alone cannot determine causal relationships.

Different causal structures may generate identical observational distributions.

Intervention provides a way to break this ambiguity by actively modifying the data-generating process and observing how the system responds.

Therefore:

Causation cannot be established through correlation alone; it must be tested through interventions on the underlying mechanisms.
